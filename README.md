# ValveVision-PiArm — คู่มือการใช้งาน

## โปรเจ็คนี้คืออะไร?

ValveVision-PiArm คือระบบ **แขนกล 6 แกน (6-DOF Robotic Arm)** ที่ทำงานบน Raspberry Pi โดยมีเป้าหมายคือ:

1. **ตรวจจับวาล์ว** ด้วย Computer Vision (กล้อง + โมเดล AI)
2. **คำนวณมุมข้อต่อ** ด้วย Inverse Kinematics (IK) — รับพิกัด (x, y, z) แล้วคำนวณว่าข้อต่อแต่ละจุดต้องหมุนกี่องศา
3. **ขยับแขนกล** ไปยังตำแหน่งที่ตรวจจับได้ ผ่านการควบคุม Servo Motor 6 ตัว

```
กล้องตรวจจับวาล์ว → ได้พิกัด (x, y, z) → IK คำนวณมุม → Servo ขยับแขนกล
```

### โครงสร้างแขนกล

```
[J6 - Gripper/มือจับ]
       |
   L4 = 180 mm
       |
[J4 - Wrist Pitch/ข้อมือก้ม-เงย]
       |
   L3 = 135 mm
       |
[J3 - Elbow/ข้อศอก]
       |
   L2 = 50 mm
       |
[J2 - Shoulder/ไหล่]
       |
   L1 = 10 mm
       |
[J1 - Base/ฐาน — หมุนซ้าย-ขวา]
```

ระยะเอื้อมสูงสุดจาก J2 = L2 + L3 + L4 = **365 mm**

### 4. INVERT — กลับทิศทางหมุน

```python
INVERT = {
    'J1': False,  # False = หมุนปกติ
    'J2': True,   # True = กลับทิศ (Servo ติดตั้งหันหลัง)
    'J3': True,
    'J4': False,
    'J5': False,
    'J6': False,
}
```

**หลักการทำงาน:**
- `False` → ส่งมุม 110° → Servo ไปที่ 110°
- `True`  → ส่งมุม 110° → Servo จะได้รับ `180° - 110° = 70°` (กลับทิศ)

> **เมื่อไหร่ต้องแก้?** หลังรัน `setup/step2_test_direction.py` แล้วเห็นว่า Servo หมุนสวนทิศที่ต้องการ → เปลี่ยน `False` เป็น `True` หรือกลับกัน

ทำตามลำดับนี้:

### Step 1: จัดตำแหน่ง Horn

```bash
python setup/step1_home.py
```

สคริปต์จะส่งสัญญาณ 90° ให้ Servo ทุกตัวแล้ว **ล็อกค้างไว้**

**ทำต่อ:** สำหรับแต่ละข้อต่อ — ถ้า Horn (ชิ้นพลาสติกบนหัว Servo) ไม่ได้ตั้งตรงที่มุม 90° ให้:
1. ถอดสกรูที่ยึด Horn
2. ดึง Horn ออกจาก Servo
3. จัดให้ตรงกับมุม 90°
4. ใส่กลับและขันสกรู

> **ทำไมต้องทำ:** Servo อิเล็กทรอนิกส์และ Horn กลไกต้องซิงค์กัน ถ้าไม่ตรง จะทำให้คำนวณมุมผิดตลอด

---

### Step 2: ทดสอบทิศทางหมุน

```bash
python setup/step2_test_direction.py
```

สคริปต์จะทดสอบทีละข้อต่อ โดยขยับจาก 90° → 110° → 90° แล้วถามว่า:

```
J1: ขยับไปทิศที่ถูกต้องไหม? (y/n):
```

- กด `y` → ทิศถูก → INVERT = False
- กด `n` → ทิศผิด → INVERT = True

**จากนั้น:** คัดลอกผลลัพธ์ `INVERT` ที่ได้ไปใส่ใน `config.py`

python setup/step3_calibrate_offsets.py

### `ik_solver.py` — Inverse Kinematics

แปลงพิกัด (x, y, z) → มุมข้อต่อทุกตัว ด้วยคณิตศาสตร์เรขาคณิต

| ฟังก์ชัน | พฤติกรรม |
|---------|---------|
| `solve_ik(x, y, z)` | คืน dict มุมทุกข้อต่อ หรือ `None` ถ้าพิกัดอยู่นอกขอบเขต |
| `solve_ik_clamped(x, y, z)` | คืนผลเสมอ โดยดึงมาที่จุดใกล้สุดถ้าเกินขอบเขต |
| `fk(j2, j3, j4)` | Forward Kinematics — เอามุมมาหาพิกัด (ใช้ตรวจสอบ) |

**Workspace ที่ใช้งานได้ดี:**
```
z=100mm → x ≈ 290–340mm
z=150mm → x ≈ 215–300mm
z=170mm → x ≈ 170–270mm
```

|-------|--------|

## ทำไมต้องแก้แขนเพื่อรองรับล้อ 19 นิ้ว

### ทฤษฎี: Reachable Workspace ของแขนกล 2-Link

แขนกลนี้ทำงานใน **ระนาบตั้ง (r–z plane)** โดยที่ J2 เป็นจุดหมุนหลัก ระยะที่ข้อมือ (J4) เอื้อมถึงได้จาก J2 คือ:

```
d_wrist = sqrt(r_w² + z_w²)

โดยที่:
  r_w = ระยะแนวนอนจากแกน J1 ถึงข้อมือ = r_target - L4
  z_w = ความสูงจาก J2 ถึงข้อมือ        = z_target - L1

เงื่อนไขที่แขนถึง:
  d_min ≤ d_wrist ≤ d_max

  d_max = L2 + L3          (แขนเหยียดตรงสุด)
  d_min = sqrt(L2² + L3²)  (แขนพับ 90° ที่ J3)
```

**ค่าปัจจุบัน:**
```
L2 = 50mm,  L3 = 135mm
d_max = 50 + 135 = 185mm
d_min = sqrt(50² + 135²) ≈ 144mm
```

### ปัญหา: Workspace ไม่ครอบล้อ 19"

ล้อ 19 นิ้วมี **radius = 241mm** วาล์วอยู่รอบขอบล้อ หมุนไปได้ทุกตำแหน่ง:

```
มุมมองจากด้านข้าง (ระนาบ r–z)

          valve บนสุด
              ●  ← z = +241mm จากจุดศูนย์กลาง
             /|\
            / | \
           /  |  \
          /   |   \
valve ←●     ⊙     ●→ valve    ← z = 0  (ระดับจุดศูนย์กลางล้อ)
  ซ้าย   จุดศูนย์     ขวา
          กลางล้อ
           \  |  /
            \ | /
             \|/
              ●  ← z = -241mm
          valve ล่างสุด

ระยะ J1 → จุดศูนย์กลางล้อ ≈ 100mm (วางชิดล้อ)
```

**Worst case** คือ valve อยู่บนสุดหรือล่างสุด (45° จากแนวนอน):

```
สมมติ J1 อยู่ระดับเดียวกับจุดศูนย์กลางล้อ และห่างออกมา 100mm:

  r_target = 100 + 241·cos(θ)
  z_target =       241·sin(θ)

กรณี valve อยู่ด้านบน 45°:
  r_target ≈ 100 + 170 = 270mm
  z_target ≈ 170mm

  r_w = 270 - 180 = 90mm
  z_w = 170 - 10  = 160mm

  d_wrist = sqrt(90² + 160²) = sqrt(8100 + 25600) ≈ 184mm  ✅ (ถึง!)

กรณี valve อยู่ด้านบนสุด (90°):
  r_target ≈ 100mm
  z_target ≈ 241mm

  r_w = 100 - 180 = -80mm  (หมายถึงแขนต้องเลย J1 ไปด้านหลัง)
  z_w = 241 - 10  = 231mm

  d_wrist = sqrt(80² + 231²) = sqrt(6400 + 53361) ≈ 244mm  ❌ (เกิน 185mm)
```

**สรุป:** ถ้า J1 อยู่ระดับจุดศูนย์กลางล้อ แขนปัจจุบันครอบ valve ได้ประมาณ **±45° จากแนวนอน** เท่านั้น

### วิธีแก้

#### ตัวเลือก 1: วาง J1 ระดับจุดศูนย์กลางล้อ (ไม่ต้องแก้แขน)

```
ก่อน:                          หลัง:
                valve              valve
                  ●                  ●
                 /                  /|
                /                  / |
               /        →         /  |
              /                   /  |
J1 ●────────                  J1 ●───┘
(ต่ำกว่า valve 378mm)        (ระดับเดียวกับ valve)
d_wrist ≈ 398mm ❌           d_wrist ≈ 184mm ✅
```

ข้อดี: ไม่ต้องสร้างแขนใหม่
ข้อเสีย: ครอบได้แค่ valve ที่อยู่ใกล้แนวนอน (±45°)

---

#### ตัวเลือก 2: เพิ่มความยาว L2 และ L3 (แก้แขน)

เพื่อให้ครอบ valve **ทุกตำแหน่งรอบล้อ** ต้องการ:

```
d_wrist_max = sqrt((r_target - L4)² + (z_target - L1)²)

worst case (valve บนสุด, J1 ระดับจุดศูนย์กลาง):
  r_target = 100mm, z_target = 241mm
  d_wrist = sqrt((100-180)² + (241-10)²)
           = sqrt(6400 + 53361)
           ≈ 244mm

ต้องการ L2 + L3 ≥ 244mm  (เผื่อ margin: ≥ 300mm)
```

**ค่าที่แนะนำ:**

| ส่วน | ปัจจุบัน | แนะนำ | เพิ่มขึ้น |
|------|---------|-------|---------|
| L2 (ต้นแขน) | 50mm | 100mm | +50mm |
| L3 (ปลายแขน) | 135mm | 200mm | +65mm |
| L2+L3 | **185mm** | **300mm** | +115mm |
| Reach รวม | 365mm | 480mm | +115mm |

```
Workspace หลังแก้ (L2=100, L3=200, L4=180):

  z=  0mm → r ≈ 80–480mm
  z=100mm → r ≈ 100–460mm
  z=241mm → r ≈ 50–400mm   ← ครอบ valve บนสุดของล้อ 19" ได้
  z=300mm → r ≈ 0–350mm
```

**หลังแก้ค่าใน config.py:**
```python
L2 = 100   # แก้ตามที่วัดจริงหลังต่อแขนใหม่
L3 = 200   # แก้ตามที่วัดจริง
```

IK solver จะคำนวณใหม่อัตโนมัติโดยไม่ต้องแก้โค้ดอื่น
